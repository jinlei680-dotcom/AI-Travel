# 提交信息中文规范

为统一仓库提交信息，约定从现在起所有 Git 提交与 PR 均使用中文撰写。推荐结构如下：

## 提交信息结构
- 标题：`<type>(<scope>): <一句话中文摘要>`
- 正文：
  - 变更说明
  - 原因与背景
  - 影响范围
  - 回归风险与验证
- 关联：关闭/关联 Issue 与相关链接
- 脚注：`BREAKING CHANGE: ...`（如存在重大不兼容变更）

## 常用 type
- `feat`：新增功能
- `fix`：缺陷修复
- `docs`：文档更新
- `refactor`：重构不涉及功能变化
- `perf`：性能优化
- `style`：样式/格式调整（不影响逻辑）
- `test`：测试相关变更
- `chore`：杂项维护（依赖、构建脚本等）

## 示例
```
feat(plan): 增加偏好弹窗与重新生成，完善卡片与筛选器

变更说明：新增偏好弹窗（目的地、日期、节奏、兴趣），调用生成接口刷新行程；统一卡片视觉并加入搜索与时段筛选。
原因与背景：提升个性化与可用性，符合计划页第一阶段目标。
影响范围：/plan 页面与相关组件，不影响其他页面。
回归风险与验证：本地预览已验证交互；如无 API 配置则走兜底逻辑。

关联：关闭 #123；设计文档更新见 docs/FrontendDesign.md。
```
